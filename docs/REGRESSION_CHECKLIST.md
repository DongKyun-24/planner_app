# Regression Checklist

이 문서는 앱 수정 후 확인할 것들을 가능한 한 빠짐없이 적어둔 체크리스트입니다.

## 자동 확인

- [ ] `npx expo export --platform android`
- [ ] `cd android && .\\gradlew.bat app:compileDebugKotlin app:processDebugMainManifest app:mergeDebugResources --rerun-tasks`
- [ ] 에뮬레이터에서 앱 실행 확인

## List / Today

- [ ] 날짜 카드 테두리가 안쪽 줄보다 조금 더 또렷하다.
- [ ] 날짜 카드 간 간격이 너무 붙거나 벌어지지 않는다.
- [ ] 날짜 헤더 아래 첫 줄이 `Task`일 때 자연스럽게 이어진다.
- [ ] 날짜 헤더 아래 첫 줄이 `일정`일 때도 빈 한 줄처럼 뜨지 않는다.
- [ ] Today 카드 안에서 `D-day / Count`가 다른 줄들과 같은 카드 안에 붙어 보인다.
- [ ] `D-day / Count -> 일정 -> 반복 일정 -> Task -> 반복 Task` 순서가 유지된다.
- [ ] 같은 묶음 안 줄선은 연하고, 묶음이 바뀌는 경계는 더 진하게 읽힌다.

## D-day / Count

- [ ] 마감일 하나를 만들면 Today에 `날짜 · 시간` 메타로 보인다.
- [ ] Count 하나를 만들면 Today에 `기준일 · D-?` 메타로 보인다.
- [ ] Count를 `100`으로 만들면 반복 기준으로 계산된다.
- [ ] Count는 10일 이내에 들어왔을 때 Today/체크에 보인다.
- [ ] D-day / Count 오른쪽 카테고리 pill이 행 가운데에 온다.
- [ ] 마감일/Count 메타 간격이 서로 같아 보인다.
- [ ] 체크 패널에서 `Task / 마감일 / Count`가 분리되어 보인다.
- [ ] 마감일 탭의 `추가`가 마감일 모드 편집창으로 간다.
- [ ] Count 탭의 `추가`가 Count 모드 편집창으로 간다.

## 메모

- [ ] 메모 탭을 누르면 바로 편집이 아니라 보기로 열린다.
- [ ] 다시 눌렀을 때 편집으로 들어간다.
- [ ] 메모를 길게 적을 때 화면이 같이 따라간다.
- [ ] 새 메모를 만들면 새 탭이 바로 보이도록 탭 영역이 이동한다.
- [ ] 마지막 메모 1개만 남았을 때 삭제가 막힌다.
- [ ] 통합 메모는 한 카드만 보이고, 탭만 바꿔가며 볼 수 있다.

## Task

- [ ] 미완료 Task가 위에 있다.
- [ ] 완료 Task가 아래에 있다.
- [ ] 체크하면 아래로 이동한다.

## 정렬 / 롱프레스

- [ ] 스크롤 중 실수로 정렬/편집 모드가 너무 쉽게 열리지 않는다.
- [ ] 길게 눌렀을 때만 의도적으로 들어간다.

## 알림

- [ ] 앱을 껐다 켜도 알림 동기화가 다시 된다.
- [ ] 백그라운드에 갔다가 돌아와도 알림이 유지된다.

## 위젯

- [ ] 리스트 위젯이 앱 리스트 화면과 톤이 크게 어긋나지 않는다.
- [ ] 달력 위젯이 앱 달력 화면과 톤이 크게 어긋나지 않는다.

## 확인 우선순위

1. Today / List
2. D-day / Count
3. 메모
4. Task
5. 알림
6. 위젯
