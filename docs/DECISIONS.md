# Decisions

나중에 다시 봤을 때 "왜 이렇게 했지?"를 바로 복구할 수 있게 기록한다.

## 2026-04-16 - Count를 마감일 D-day와 분리

- Count는 시작일 기준 기능이다.
- 시작일을 `1일`로 계산한다.
- 입력 방식은 `중요 D-day 주기`에 `100, 365`처럼 넣는 방식으로 정리한다.

이유:
- 마감일 D-day와 Count는 계산 방식이 다르다.
- 두 기능을 같은 개념으로 붙여두면 UI와 알림 규칙이 계속 충돌한다.

## 2026-04-16 - Today 내부 순서 고정

- `D-day / Count -> 일정 -> 반복 일정 -> Task -> 반복 Task`

이유:
- 먼저 봐야 하는 정보와 실행 정보가 섞이지 않게 하기 위함이다.
- 같은 종류끼리 묶여 있어야 시각적으로 읽기 쉽다.

## 2026-04-16 - Count는 10일 이내일 때만 Today/체크에 노출

- Count는 항상 보이는 것이 아니라 다음 기준일까지 10일 이내일 때만 눈에 띄게 보여준다.

이유:
- 평소에도 항상 보이면 마감일 D-day보다 더 시끄러워진다.
- 가까워졌을 때만 강조하는 쪽이 실제 사용감에 맞는다.

## 2026-04-16 - 메모는 보기와 편집을 분리

- 탭을 누르면 먼저 보기로 연다.
- 편집은 한 번 더 눌렀을 때 들어간다.

이유:
- 실수로 편집 상태에 들어가는 일이 줄어든다.
- 읽기와 수정의 의도를 분리하는 편이 안정적이다.

## 2026-04-16 - Task는 미완료 우선 정렬

- 미완료 위
- 완료 아래

이유:
- 지금 해야 할 일을 먼저 보게 해야 하기 때문이다.

## 2026-04-16 - Today의 D-day / Count는 카드 안 row로 취급

- D-day / Count를 위에 떠 있는 별도 블록처럼 두지 않는다.
- 같은 날짜 카드 안의 첫 줄처럼 보이게 한다.

이유:
- 따로 뜨는 구조에서는 선, 간격, 경계가 계속 어색해졌다.
- 같은 리스트 구조 안에 넣어야 디자인을 통일하기 쉽다.
