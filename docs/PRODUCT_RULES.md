# Product Rules

이 문서는 앱이 사용자에게 어떻게 보여야 하는지 정리합니다.

## 공통 개념

### Count와 마감일 D-day는 다르다

- `마감일 D-day`는 특정 날짜까지 남은 일수를 보는 기능이다.
- `Count`는 시작일 기준으로 반복되는 기준일을 보는 기능이다.
- Count는 시작일을 `1일`로 계산한다.
- Count는 마감일 D-day 옆에 붙은 보조 기능이 아니라 별도 개념으로 취급한다.

### Count 기준

- `중요 D-day 주기`에 쉼표로 여러 숫자를 넣을 수 있다.
- 예: `100, 365`
- 의미:
  - `100일, 200일, 300일 ...`
  - `365일, 730일, 1095일 ...`
- Count는 다음 기준일까지 `10일 이내`일 때 Today와 체크 패널에서 보인다.

## Today / List 규칙

### Today 카드 내부 순서

Today 카드 안의 순서는 아래를 유지한다.

1. `D-day / Count`
2. `일정`
3. `반복 일정`
4. `Task`
5. `반복 Task`

### 날짜 카드 규칙

- 날짜 카드 바깥 테두리는 안쪽 줄선보다 조금 더 또렷해야 한다.
- 날짜 카드 간 간격은 너무 붙지 않되 과하게 벌어지지 않아야 한다.
- 날짜 헤더 아래 첫 줄은 `Task`일 때와 `일정`일 때 모두 자연스럽게 이어져 보여야 한다.

### D-day / Count 행 규칙

- D-day와 Count는 Today 카드 안의 다른 줄들과 같은 카드 안에서 이어져 보여야 한다.
- 예전처럼 위에 따로 뜨는 위젯처럼 보여서는 안 된다.
- 오른쪽 카테고리 pill은 행 높이 기준 세로 가운데에 와야 한다.
- 내용 아래 메타 줄 간격은 마감일과 Count가 같아야 한다.
- 메타 형식:
  - 마감일: `날짜 · 시간`
  - Count: `기준일 · D-?`

## 체크 패널 규칙

- 체크 패널은 `Task / 마감일 / Count` 3분할로 간다.
- 각 탭은 서로 다른 성격의 항목만 보여준다.
- `추가` 버튼은 현재 탭 기준으로 동작해야 한다.
  - Task 탭이면 Task 추가
  - 마감일 탭이면 마감일 추가
  - Count 탭이면 Count 추가

## 메모 규칙

- 메모 탭을 누르면 먼저 `보기`로 열린다.
- 편집은 한 번 더 눌렀을 때 들어간다.
- 새 메모를 만들면 새 메모 탭이 바로 보이도록 탭 스크롤이 이동해야 한다.
- 메모가 1개만 남은 상태에서는 삭제할 수 없다.
- 통합 메모는 한 카드만 보이고, 위 탭을 바꿔가며 보는 구조를 유지한다.

## 알림 규칙

- 일정이 많아도 앱 재진입 후 알림 동기화가 다시 되어야 한다.
- Count와 마감일은 같은 방식으로 보이지 않더라도, 둘 다 Today/체크에서 놓치지 않게 노출되어야 한다.

## 위젯 규칙

- 리스트 위젯, 달력 위젯은 앱 화면의 구조와 톤을 최대한 따라간다.
- 앱 본문과 위젯이 전혀 다른 제품처럼 보이면 안 된다.
